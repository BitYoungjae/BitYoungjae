업무 밖에서는 같은 시선으로 AI 시스템, 개발자 도구, Linux 환경을 파고들고 있습니다.

## 핵심 역량

- 빠른 독학: 지금 다루는 모든 언어와 기술을 혼자 공부해서 익혔습니다. 강의보다 공식 문서를 직접 파고드는 걸 좋아하고, 새로운 도메인이든 패러다임이든 직접 만들면서 빠르게 소화합니다.
- 구조로 풀기: 얽힌 비즈니스 규칙, 불투명한 모델 동작, 문서 없는 인터페이스를 — 상태가 명확하고, 경계가 분명하고, 안이 들여다보이는 시스템으로 바꿉니다.
- 직접 확인: 시스템이 어떻게 동작하는지 짐작하지 않습니다. 관찰하고, 실험하고, 확인합니다. 안이 보이지 않으면 볼 수 있는 도구를 직접 만듭니다.
- 딱 맞는 설계: 문제 크기에 맞게 만듭니다. 런타임 검사보다 타입 제약. 프레임워크보다 집중된 모듈. 쓸데없는 유연성보다 의도된 제한.
- 불편하면 만든다: 같은 불편이 두 번 느껴지면 도구를 만들어 없앱니다 — CLI, 자동화 파이프라인, 에디터 확장.

아직 작업 중이라 공개하기는 어렵지만, LLM을 애플리케이션 레벨에서 어떻게 쓸지, 프로덕션 등급의 에이전트 시스템은 어떻게 만들 수 있을지 고민하고 연구하면서 이것저것 만들어 보고 있습니다.

최근에는 Claude Code 내부를 들여다보면서 감을 잡아가고 있는데(antml, Auto Memory, Sytem Reminder 등등.. 흥미로운 지점들이 많아요), 알면 알수록 대단한게 제약투성이인(거의 모래성에 가까운) 기반 위에서 깊은 고민과 영리한 설계로 이만큼의 가치를 뽑아낸게 정말 대단하다고 느껴집니다.

## 일하는 방식

- 설계가 먼저, 구현은 그 다음 — 아키텍처 문서와 타입 정의가 코드에 앞섭니다
- AI 시스템을 프로덕션 인프라로 다룹니다: 명시적 가정, 신뢰 경계, 운영 런북
- 한 번 쓰고 버리는 기교보다 다시 쓸 수 있는 구조를 선호합니다
- 트레이드오프가 명확한 실용적 아키텍처를 선택합니다
- 결정과 검증 과정을 문서로 남겨서 누구든 이어갈 수 있게 합니다
